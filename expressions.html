Operators:

Heck supports the operators that are common among the C-alike languages. Most of them can be overloaded.

Here is a list of every operator in Heck, grouped in order of precedence:

1:

Operator Name Syntax
++ postfix increment a++
-- postfix decrement a--
typeof() type of typeof(a)
() function call a(...)
[] array subscripting a[b]
?[] null conditional array subscripting a?[b]
. member access a.b
?. null conditional member access a?.b

2:

Operator Name Syntax

-- prefix decrement --a
- unary minus -a
! logical NOT !a
~ bitwise NOT (one's complement) ~a
as type cast a as type

3:

Operator Name Syntax

/ division a / b
% modulo (remainder) a % b

4:

Operator Name Syntax
+ addition a + b
- subtraction a - b

5:

Operator Name Syntax

6:

Operator Name Syntax

7:

Operator Name Syntax

8:

Operator Name Syntax

9:

Operator Name Syntax
< less than a < b

10:

Operator Name Syntax

Operator Name Syntax

15:

Operator Name Syntax
= direct assignment a = b
+= assignment by sum a += b
-= assignment by difference a -= b
*= assignment by product a *= b
/= assignment by quotient a /= b
%= assignment by remainder a %= b
<<= assignment by bitwise left shift a <<= b
>>= assignment by bitwise right shift a >>= b
&= assignment by bitwise AND a &= b
^= assignment by bitwise XOR a ^= b
|= assignment by bitwise OR a |= b

*Note: In Heck, the bitwise operators & ^ and | have higher precedence than the equality operators. This is different from the precedence order in C and C++ among other languages, which has been subject to criticism because it basically forces the user to put parentheses around bitwise operations in order to make any useful comparisons.
