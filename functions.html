Functions

Functions are blocks of code that can be run elsewhere multiple times, and can return values

// a function being created

func lol() {
    // code
}

// calling that function

lol()

Function parameters have the same syntax as variable declarations.

func test(a, b) {
    // code
}

You can give function perameters different types. (as seen in variables)

You return values like this:

func add(a: int, b: int) {
    return a + b
}

add(10, 5)

This isn't required with all functions, but you can return an explicit data type like this:

func add(a: int, b: int) -> int {
    return a + b
}

add(5, 7)

Function overloading allows you to create multiple functions with the same name, but they need different perameters to distinct the function calls.

func add(a: int, b: int) {
   return a + b
}

func add(a: float, b: float) {
    return a + b
}

// calling the int function
add(4, 6)

// calling the float function
add(1.0, 2.3)

Generic functions are functions with not specified perameters. This allows functions to assume types.

func add(a, b) {
    return a + b
}

// calls the function as an int
add(2, 5)

// calls the function as a float
add(0.4, 2.0)

Function overloading takes priority over generic functions.

func add(a, b) {
    return a + b
}

func add(a: int, b: int) {
    return a + b
}

// calls the second function
add(3, 4)

// calls the first function with a different data type
add(1.0, 3.5)

External declerations allow you to declare a function without the function body.

class Foo {
    let value: int = 0
    // function is declared her but defined elsewhere
    // this makes the class definition easier to read
    extern func printValue()
}

// the function definition that coincides with the extern declaration
func Foo.printValue() {
    print(value)
}

let myFoo = Foo() // creates "Foo" object
myFoo.printValue() // prints 0
